SQL Server

使用 SQL Server 团队构建的映像，您可以在几分钟内轻松设置 SQL Server。您可以使用针对快速开发/测试环境提供的免费 MSDN 许可证创建虚拟机，也可以使用 SQL Server AlwaysOn 跨多个 Windows Azure 区域部署复杂的生产应用程序。

开放，提供更多选择

您可以在 Windows Azure 上部署全套开放的以及社区驱动的操作系统以及软件解决方案。借助 Windows Azure，您可以选择 Ubuntu 和 SUSE 等各类 Linux 发行版， Windows Azure 是开放的，为您提供了很多选择。

省钱，获得更高灵活性

针对快速计算型工作负载、快速开发和测试环境，以及频繁停止和启动等场景，在 Windows Azure 中部署虚拟机时，您将按分钟付费。包括针对 Windows、Linux和SQL Server的计费皆是按分钟收取。您甚至可以让整个平台根据您的计划或实例的 CPU 使用率自动缩放。我们将提供能满足您需要的基础结构和软件计费模型，而不是为了满足我们的需要！

混合连接

使用虚拟网络，您可以控制和配置您网络的方方面面，定义子网和首选 DNS IP。您可以通过互联网，用安全的方式通过 VPN 在 Windows Azure 中连接您的虚拟机。

此服务包含跨多个实例的负载平衡，且易于配置。您可以创建负载平衡器，添加安全 ACL 以便控制访问权限，并定义具体的探测，以便帮您监视应用程序运行状况，所有这些都可以在易于使用的门户中直接进行。

更快、更大且更好的 D 系列虚拟机

您现在可以部署更大的 D 系列虚拟机，满足您的工作负载的需要。这种大小的虚拟机运行速度更快，各方面的指标更强大，处理器速度更快、内存更大，提供基于 SSD 的存储所支持的临时磁盘，方便虚拟机进行极快速的临时存储。快来试一试吧！今天就可以体验。

您可以利用 Windows Azure 虚拟机将 Windows Server、Linux 或第三方软件映像部署到 Windows Azure，而且可以选择库中的映像或使用自己的自定义映像。

我们按分钟对虚拟机收费，按小时费率列出价格，并且当您的 VM 的运行时间不足 1 小时，我们将按总分钟数计费。

为满足不同客户的需求，我们为 Windows 和 Linux VM 引入了两个新计算级别：“基本”和“标准”。

“基本”计算级别：这一新的计算实例级别在配置上类似于“标准”级别，但价格较低。这些实例不包含负载平衡器和自动扩展功能。它们非常适用于可能不需要上述功能的单实例生产应用程序、开发用工作负载、测试服务器和批处理应用程序。“基本”计算级别目前仅适用于“通用”实例。目前“基本”实例处于预览阶段。

“标准”计算级别：这一级别的计算实例提供适合运行大量应用程序的计算、内存和 IO 资源的最优组合。这些实例包含自动扩展、负载平衡和内部负载平衡功能，且不需另外收费。内部负载平衡功能目前处于预览状态。“标准”计算级别对于“通用”、“内存密集型”实例均适用。此处提供实例的详细配置。

Windows Azure 映像库提供下列版本的 Windows Server ： Windows Server 2008, Windows Server 2008 R2, Windows Server 2012 及 Windows Server 2012 R2。

Windows Azure 映像库也提供下列版本的 SQL Server: 包含 SQL Server 2014, SQL Server 2012, SQL Server 2008 R2 各版本。

提醒：

Windows Azure 虚拟机磁盘中包含一个临时盘， D:磁盘 (Windows) 或者 /dev/sdb1 (Linux)。他们仅提供临时存储，所以可能会有丢失数据的风险且数据无法恢复。该情况发生的常见方式是虚拟机迁移到不同主机上时。虚拟机迁移的一些原因包括调整虚拟机大小、更新主机或主机上硬件故障。

常见问题


 如果将我的本地 Windows Server 映像上载到 Windows Azure，是否需要提供我的许可证密钥？

不需要。Microsoft 将为任何正在运行的实例提供 Windows Server 许可证密钥。

 如果我将在 Windows Azure 上运行的我的 Windows Server 映像下载到本地，Windows Azure 中提供的许可证是否可以转移到本地？

不可以。如果您将在 Windows Azure 上运行的 Windows Server 映像下载到本地，则需要为此映像提供许可证。

 我是否需要单独购买用于虚拟机的本地磁盘存储？

当您创建虚拟机时，Windows Azure 中的所有虚拟机至少配置了两个磁盘 - 一个是操作系统磁盘，另一个是临时本地磁盘（有时称为资源磁盘）。临时本地磁盘存储区随虚拟机一起提供，不单独收费。操作系统磁盘按磁盘的常规费率收费。若要浏览虚拟机配置，请访问此处。

 虚拟机实例价格是否包括与在虚拟机中运行的映像关联的存储费用？

虚拟机实例价格不包括与在虚拟机中运行的映像关联的任何存储费用。

 什么是内部负载平衡？

我们正在使用专用 IP 地址对 Windows Azure VM 进行负载平衡的功能。内部负载平衡的终结点将只能在虚拟网络内（在虚拟机处于 VNET 内部的情况下）或云服务内（在虚拟机未处于 VNET 内部的情况下）访问。在创建某些应用程序级别并不公开但要求负载平衡功能的多级应用程序时，内部负载平衡将会很有用。

 如果我的已部署实例处于“已停止”状态，是否仍向我计费？

如果您的实例处于“已停止取消分配”状态，将不对它计费。如果您的实例处于“已停止分配”状态，因为它仍占用计算资源，将对分配的虚拟内核计费，但不对软件许可证本身计费。

假定一个计算实例处于以下状态，其计费情况供您参考：

状态计费详细信息
正在启动 是 VM 的初始启动状态，此时它们正在经历启动周期。此期间将计费，因为 VM 正在运行。
正在运行（已启动） 是 VM 的正在运行状态。
已停止 否 如果 VM 已停止（无论是已分配还是已取消分配），将不对软件计费。对于已分配的已停止 VM，仍将对内核计费。
已删除 否 VM 已删除，不再占用内核。
 如何确保在已停止 VM 后不对其计费？

若要确保不计费，请从管理门户停止 VM。您还可以通过 Powershell 调用“PostShutdownAction”等于“StoppedDeallocated”的 ShutdownRoleOperation 来停止 VM。不过，如果从内部（例如，使用 Windows 中的电源选项）或通过 PowerShell 调用“PostShutdownAction”等于“Stopped”的 ShutdownRoleOperation 来关闭 VM，则将继续计费。

 如何检查是否已正确停止 VM 并且将不对其计费？

您可以在“虚拟机”选项卡和 VM 仪表板中检查 VM 的状态。如果 VM 状态为“已停止”，则仍计费。仅当状态为“已停止(已取消分配)”时才表明 VM 不再产生费用。例如，如果使用 Windows 中的电源选项停止 VM，该 VM 仍将产生费用；但如果从管理门户停止 VM，该 VM 将不再产生任何费用。

 如果我的虚拟机部署了不到一小时，将会向我收费多少？

按 VM 的部署时间长短对虚拟机计费。如果 VM 的部署时间不到 1 小时，或者为多个小时加上 1 小时的一部分，将仅对该持续时间收费。Windows Azure 不会将不到 1 小时的时间向上舍入为完整的 1 小时来进行计费 - 您只需为 VM 处于“正在运行”状态的该持续时间段付费。

 如何对不同的实例大小计费？

在显示给您的帐单上，所有小时数都会转换为小型实例小时数。

 如果我有位于不同区域的两个实例，如何对这两个实例之间的数据传输收费？

对每个实例跨区域的数据传出收费。

 我何时能够使用 Windows Azure SQL Database 而不是虚拟机中的 SQL Server？

如果您有现有应用程序或工作负荷，只想利用并移到云，虚拟机中的 SQL Server 是获得云的好处的最简单方式。如果您想生成在云中运行的新关系应用程序或将本地应用程序扩展到云，SQL Database 是最佳的长期解决方案。有了虚拟机中的 SQL Database 和 SQL Server，开发人员就可以利用常见工具和技术在业务需求变化时将应用程序移到本地或重新移回云。

 SQL Server 2012 Standard Edition 与 SQL Server 2012 Web Edition 相比有哪些优势？

SQL Server 2012 Standard Edition 与 SQL Server 2012 Web Edition 相比具有几个优势。例如，Standard Edition 允许您运行功能强大的 SQL Server Analysis Services 并支持高可用性方案（如故障转移群集以及数据库镜像和备份压缩）。此文章提供更多详细信息。

 SQL Server 2012 Enterprise Edition 与 SQL Server 2012 Standard Edition 相比有哪些优势？

SQL Server 2012 Enterprise Edition 在几个关键功能方面具有优势，例如数据库引擎、Analysis Services 和 Reporting Services。此外，Enterprise Edition 也支持高可用性和灾难恢复 (HADR) 方案。此文章详细介绍了功能差异。

 通常的 SQL Server Enterprise Edition 与面向数据仓库的 SQL Server Enterprise 版本有何不同？

数据仓库是具有代表性的复杂工作负载，为实现最佳查询性能和硬件（CPU、IO 和存储）平衡而设置、配置和调优。借助此映像，Microsoft 使承载于 Windows Azure 之上的 SQL Server Enterprise 的广泛工程设计得以完善，从而在云中实现最佳数据仓库配置。利用多年来 SQL Server Enterprise 认证第三方硬件的最佳做法，此映像为 Windows Azure 虚拟机中的 SQL Server 提供了同样的专业技术。

Blob、表、队列和文件

Azure 存储服务在数据存储和检索方面具有较高灵活性，可以用于存储大量非结构化数据方面，例如使用 Azure Blob 存储文档和媒体文件；或者使用 Azure 表存储结构化 nosql 数据；或者使用 Azure 队列存储可靠消息；以及使用基于 SMB 的 Azure 文件服务将本地应用程序迁移到云中。

高级存储

Windows Azure 高级存储为在 Windows Azure 虚拟机上运行的 I/O 密集型工作负荷提供高性能、低延迟的磁盘支持。你可以向虚拟机 (VM) 附加几个高级存储磁盘。借助高级存储，应用程序对于每个 VM 可以具有多达 32 TB 的存储，对于每个 VM 可以实现 50,000 IOPS（每秒输入/输出操作数），并且读取操作的延迟非常低。高级存储目前仅可用于在 Windows Azure 虚拟机所使用的磁盘上存储数据。

高伸缩性

Azure 存储服务可随时满足你不断提高的数据需求，最多可为每个帐户提供 200 TB 存储。一个订阅最多支持 50 个存储帐户，因此可提供 PB 级别的存储，您可将其用于实现最大规模的部署方案。无论构建消费产品网站，或是 TB 级的大数据应用程序，Azure 都能应付自如。

耐久且高度可用

Azure 存储服务会自动复制你的数据，帮助防范意外硬件故障，确保数据随时可用。我们将在一个区域中保留 3 个副本。地域冗余选项可在数百公里外的区域额外创建 3 个副本，进一步提高可靠性并实现灾难恢复。地域冗余备份可读访问选项，可将数据可用性进一步提高至99.99%。

专为开发人员设计

借助对 .NET、Java、Android、C++ 和 Node.js 等常见客户端库的支持，自由构建应用程序。Azure 存储服务中的数据还可以通过 REST API 访问，任何能够发起 HTTP/HTTPS 请求的语言都可调用该 API。Azure 存储服务提供可靠的一致性保障，简化了云应用程序的开发，使构建在 Azure 之上的应用程序具备可预测的体验。

全球覆盖

Azure 存储服务已在全球建立19个数据中心，其中包括中国东部以及中国北部。你可以将文件放置在对业务最有利的区域，还可以根据需要扩展存储服务或跨数据中心部署，并将数据保留在靠近客户的地理位置，让用户实现快速高性能访问。

Blob、表、队列和文件

Azure 存储服务在数据存储和检索方面具有较高灵活性，可以用于存储大量非结构化数据方面，例如使用 Azure Blob 存储文档和媒体文件；或者使用 Azure 表存储结构化 nosql 数据；或者使用 Azure 队列存储可靠消息；以及使用基于 SMB 的 Azure 文件服务将本地应用程序迁移到云中。

高级存储

Windows Azure 高级存储为在 Windows Azure 虚拟机上运行的 I/O 密集型工作负荷提供高性能、低延迟的磁盘支持。你可以向虚拟机 (VM) 附加几个高级存储磁盘。借助高级存储，应用程序对于每个 VM 可以具有多达 32 TB 的存储，对于每个 VM 可以实现 50,000 IOPS（每秒输入/输出操作数），并且读取操作的延迟非常低。高级存储目前仅可用于在 Windows Azure 虚拟机所使用的磁盘上存储数据。

Windows Azure 存储是一类可供您随时随地访问的高度持久、高度可用且高度可缩放的云存储。

根据存储容量、存储事务数（对存储执行的读写操作的次数）和已传输的数据向客户收取 Windows Azure 存储空间的费用。

存储分为四类：

块 Blob：块 Blob 是一种用于可靠地存储大量 Blob 数据的经济实用的可扩展方式。典型用例包括数据共享、大数据和备份。

页 Blob 和磁盘：页 Blob 提供针对随机访问和频繁更新优化的块存储。由于其性能、持久性和可靠性较高，因此可将其用作 Windows Azure 中的 VM 的永久性磁盘。

表和队列：表是高度可缩放且极易使用的 NoSQL 数据库。它们为结构化数据和半结构化数据提供基于存储的密钥值。队列提供了一个用来计划异步任务的可靠的消息传递系统，并实现了永久性消息传递。

文件预览版：文件允许Windows Azure 数据中心的 VM 使用 SMB 协议安装共享文件系统，以及使用标准 Windows 文件 API 访问文件系统。虚拟机可以同时连接到这些文件系统，这使得您可以轻松地在不同角色和实例之间共享永久性数据。您还可以使用 Windows 文件 API 或文件 REST API 访问您的数据。

另外，我们提供了存储的三个冗余级别：

本地冗余存储 (LRS)：我们在一个区域内维护数据的多个副本以提供高持久性。

地域冗余存储 (GRS)：我们通过在同一地域内相隔数百英里的两个区域之间进行复制来异步提供额外的数据持久性。在两个区域中，我们维护数据的多个副本。

读取访问地域冗余存储 (RA-GRS)：除了地域冗余的存储服务外，我们提供对辅助区域（它具有主存储系统中数据的最终一致的版本）中存储帐户的只读访问。当主区域中的存储帐户不可用时，客户可以使用此服务访问其数据。

高级存储

高级存储是基于高性能固态硬盘 (SSD) 的存储，旨在支持 I/O 密集型工作负荷，具有非常高的吞吐量和低延迟。借助高级存储，可以设置永久磁盘，并按照应用程序要求配置其大小和性能特征。

页 Blob 和磁盘

页 Blob 高级存储当前采用三种磁盘大小提供：P10 (128 GB)、P20 (512 GB) 和 P30 (1,024 GB)。这些磁盘大小提供不同的每秒输入/输出操作数 (IOP)、吞吐量限制和每 GB 每月价格，从而可实现简化并更好地预测价格。可选择最符合应用程序的所需存储大小、IOP 和吞吐量的选项。可以将若干个永久磁盘附加到 VM 并对于每个 VM 配置最多 32 TB 的存储，每个 VM 有超过 50,000 的 IOPS，读取操作的延迟少于 1 毫秒。高级存储由一种新的 DS 系列的虚拟机进行支持。DS 系列的虚拟机专为高级存储而设计，可以进一步提高磁盘的读写性能。DS 系列虚拟机的定价和计费标准与 D 系列虚拟机相同。

常见问题


 如果我只是将存储服务用于一个月的几天，是否按比例分摊成本？

是。存储容量按一个月内每日的平均数据存储量 (以 GB 为单位) 计费。例如，如果您在前半个月持续使用了 10 GB 的存储量，而在后半个月未使用任何存储量，则将按 5 GB 的平均存储使用量进行计费。

 我如何知道我的数据是在读访问的地域冗余存储服务、地域冗余存储服务还是本地冗余存储服务中？

默认情况下，存储帐户设置为地域冗余。您可以通过 Windows Azure 管理门户或以编程方式通过服务管理 API 将这些设置更改为读访问的地域冗余或本地冗余。您还可以在门户上或使用服务管理 API 查看其状态。

 RA-GRS 的计费周期是如何运行的？

启用 RA-GRS 的情况下，自 RA-GRS 启用之时起将向存储帐户收取至少 30 天的 RA-GRS 费用。例如，如果 RA-GRS 于 2014 年 5 月 1 日启用并于 2014 年 5 月 5 日 禁用，则将向存储帐户收取直到 2014 年 5 月 30 日的 RA-GRS 费用，此后将实行 GRS 定价。如果 RA-GRS 于 2014 年 5 月 1 日启用并于 2014 年 6 月 14 日 禁用，则将收取直到 2014 年 6 月 14 日的 RA-GRS 费用，2014 年 6 月 15 日将实行 GRS 定价。

 如果我将空的 100GB 磁盘附加到我的虚拟机，是会向我收取整个 100 GB 的费用还是只收取我使用的存储空间的费用？

如果空的 100GB 磁盘由高级存储帐户提供支持，则将向你收取 P10 (128GB) 磁盘的费用。如果使用任何其他类型的存储帐户，则只会对用于存储写入磁盘的数据的存储空间计费，而无论分配的磁盘大小。

构建支持大规模的 SaaS 应用程序

构建软件即服务 (SaaS) 应用程序的开发人员可利用 SQL 数据库提供灵活性以支持爆炸性增长和盈利的商业模型。对于具有不可预测的数据库资源消耗的工作负载，弹性数据库模型可使您在一组数据库之间均分要利用的资源，无需超量配置数据库以满足峰值需求，反之，您可以使用弹性数据库池来让成百或成千的数据库利用您控制预算内的资源。借助可跨数据库组控制价格和性能的购买模型，您可以推动成本效益。

轻松管理大量数据库

当您扩展到数以千计的数据库时，简单的管理任务也会变得复杂，这是爆炸性增长带来的挑战。随着业务不断增长，SaaS 开发人员将花费大量时间编写复杂逻辑以应对这些架构变化以及其他管理运营问题。使用弹性数据库，生活变得更轻松。您选取一个脚本，将它作为作业提交，Azure SQL 数据库将完成跨多个数据库执行脚本的剩余工作。弹性数据库池可以简化跨许多数据库缩放的应用程序的生成和管理，因此，生成针对单个数据库或上千数据库的应用程序就和使用熟悉的 T-SQL 和 ADO.NET 编程模型一样轻松。您还可以运行集中式查询操作，例如跨多个数据库的报告和数据提取、返回单个统一结果集。

支持弹性扩展，提供更可预测的性能

在 Windows Azure 强大功能的支持下，高吞吐量应用程序可以在上千个数据库之间灵活地扩展。新增的 SQL 数据库服务级别使应用程序能够轻松地在每个数据库上扩展或收缩，获得可预测的性能。Windows Azure SQL 数据库的弹性扩展预览版可简化应用程序的开发与管理，并可在大量数据库之间横向扩展。

常见问题

SQL 数据库

 我是否可以将现有数据导入或迁移到 Azure SQL 数据库？

可以。使用常见的工具和开发框架，我们保证可以轻松将现有 SQL Server 应用程序的数据层和数据库数据迁移到 Azure SQL 数据库，并提供各种功能强大的工具，几乎可以适应任何场景。请参阅 SQL 数据库迁移概述获取相关选项和其他信息。

单个数据库

 “基本”、“标准”和“高级”级别数据库的使用情况如何体现在我的账单上？

对于“基本”、“标准”和“高级”级别，我们基于版本和性能级别提供统一的可预测日费率对你收费。实际使用量是每小时按比例计算的，因此你的账单可能会显示天的分数形式。例如，如果某个数据库在一个月内存在了 12 小时，则你的账单将显示 0.5 天的使用量并按 0.5 天计费。此外，性能级别（例如“基本”、S1 和 P2）会在账单中进一步细分，以便你更方便地了解在一个月中你在每个性能级别上使用数据库的天数。

 如果我的数据库活跃时间不足一个小时如何计费？

将针对数据库存在的每个小时向你收费，不考虑其使用量，也不考虑其活跃时间是否不到一小时，将使用该小时内应用的最高费率。例如，如果你创建了一个数据 库，5 分钟后删除了它，则将按该数据库存在 1 小时收费。如果你创建了一个基本数据库并立即将其升级为标准版 S1，则第一小时将按标准版 S1 费率收费。如果你删除了一个数据库，之后使用相同名称创建了另一数据库，在该小时内将对两个独立数据库分别收费。

  我如何设置单个“基本”、“标准”或“高级”数据库？

可以在任意服务器上创建“基本”、“标准”和“高级”数据库。你可以在任何两个版本之间进行数据库的升级或降级，而无需使数据库脱机。

  更改单个数据库的服务级别或性能级别需要多长时间？

更改数据库的服务级别可能需要在内部复制数据库。当在“标准”版或“高级”版之间切换或更改“标准”或“高级”数据库的性能级别时，可能进行这样的复制。如果进行复制，可能需要几分钟至几小时的时间，具体取决于数据库的大小。如果你在创建数据库后立即更改它的版本或性能级别，则与升级已填充数据的数据库相比，升级速度将更快。

  什么是单个数据库吞吐量单位 (DTU)？

SQL 数据库引入了数据库吞吐量单位 (DTU)，以便为客户提供预测性更强的性能体验。DTU 代表数据库引擎的性能，是一个合并了 CPU、内存以及读取和写入速率的度量单位。此度量单位可以帮助客户评估“基本”、“标准”和“高级”数据库提供的六个 SQL 数据库性能级别（基本、S1、S2、P1、P2 和 P3）的相对性能。有关详细信息，请访问 MSDN 上的性能指南。

  更改单个数据库的版本或性能级别后多久计费费率会发生相应变化？

“基本”、“标准”和“高级”数据库基于小时内采用的最高版本和性能级别按小时计费。更改版本或性能级别后，新费率在更改完成后立即生效。例如，如果你在晚上 10:00 将数据库从“基本”版升级到“高级”版，并且升级过程在第二天下午 1:00 完成，则将仅从升级完成的那一小时开始按“高级”版费率向你收费。如果你在上午 11:00 将数据库从“高级”版降级到“标准”或“基本”版，并且降级在 14:00:00 完成，则在当天直到在 14:00:00 点降级完成时将按“高级”版费率对数据库收费，然后将按“标准”或“基本”版费率对数据库收费。

  如何更改单个“标准”或“高级”数据库的性能级别？

在 Azure 管理门户中，可使用数据库“缩放”选项卡在你修改“标准”或“高级”数据库性能需求时设置性能级别。

  可以按何种频率更改单个数据库的版本或性能级别？

更改数据库的版本或性能级别是需要周密思考、慎重对待的操作。在 24 小时内，最多允许更改数据库的版本或性能级别 4 次。在 Web 和企业版之间切换不在此限制之列。

 每小时、每分钟和每秒的事务速率对于性能目标意味着什么？

每种性能级别被设计为相对上一级别性能逐渐提高。每小时、每分钟和每秒速率定义了客户的应用程序和吞吐量需求所要求的一种精度级别。例如，“基本”级别设计 用于需要每小时处理一定事务数的应用程序。“标准”级别设计用于性能要求更高的应用程序，此时指定每分钟处理一定事务数比较合适。“高级”级别针对大多数 关键任务吞吐量设计，其中支持大量并发事务要求使用每秒事务数表示速率。通过使用小时、分钟和秒，客户可以快速了解哪个级别（如“基本”）最符合自己吞吐量的要求。了解有关SQL 数据库性能目标 的更多信息。

 标准异地复制的成本是多少？

标准异地复制是由世纪互联管理的选择加入的异地复制功能。选择加入一个数据库后，将会在同一地理区域内至少 500 英里外的预配对 Azure 区域中创建一个辅助数据库。选择加入标准异地复制会创建与主数据库性能级别相同的脱机辅助数据库，收取的费用是主数据库的 75%；例如，如果你的主数据库的性能级别为 S2，收费为每月¥651，则你的 S2 脱机辅助数据库的收费将为每月 ￥488.25，总计每月 ￥1,139.25。主数据库和脱机辅助数据库之间的异地复制产生的流量费用包含在脱机辅助数据库的费用内。

 活动异地复制的成本是多少？

活动异地复制是由客户管理的选择加入的异地复制功能。你可选择在任何 Azure 区域中创建最多 4 个联机（可读）辅助数据库。你将按照与主数据库相同的费率为创建的活动异地复制辅助数据库付费；例如，如果你的主数据库的性能级别为 P1，收费为每月 ¥4,030，则你的 P1 可读辅助数据库的收费将为每月 ¥4,030 ，总计每月 ￥8,060。主数据库和联机辅助数据库之间的异地复制产生的流量费用包含在联机辅助数据库的费用内。

  利用审核功能将对我们的账单产生什么影响？

审核功能是 SQL 数据库服务的内置功能，无需另行付费，“基本”、“标准”和“高级”版均提供此功能。但是，为了存储审核日志，审核功能会利用你指定的 Azure 存储账户。表格和队列的 Azure 存储费率将取决于你的审核日志的大小。

  如果我使用自动导出，是否将额外向我收费？

许多客户都使用自动导出来定期创建数据库的逻辑备份。“基本”、“标准”和“高级”版数据库都具有对时点还原的内置支持，在许多情况下降低了使用自动导出的 需求。自动导出在同一服务级别中创建数据库的副本。此附加数据库在导出操作持续期间一直存在并且在它存在时按普通的小时费率收费。取决于你导出数据库的频 率和数据库的大小（数据库越大，复制和导出所需的时间越长），你可以显著提高被导出的数据库的成本效益。

  Azure SQL 数据库代表我管理什么？

作为一种关系数据库服务，Azure SQL 数据库避免了管理本地关系数据库和硬件的成本和涉及的复杂性。作为该服务的一部分，我们取得、设置和管理物理硬件，安装所需的软件和修补程序并通过 2 个本地副本维护一个高可用性级别，而不会导致任何额外的成本。

  我何时应使用 Azure SQL 数据库而不是虚拟机中的 SQL Server？

如果您有现有应用程序或工作负荷，只想利用并移到云，虚拟机中的 SQL Server 是获得云的好处的最简单方式。如果您想更新和生成在云中运行的新关系应用程序或将本地应用程序扩展到云，SQL 数据库是最佳的长期解决方案。

 是否有维护时间段，在 Windows 计划的软件维护期间我的数据库实例是否可用？

通过 SQL 数据库内置的高可用性技术，客户不会因为常规软件维护而发生停机。

 我是否要启用备份还是让系统自动备份？

Azure SQL 数据库会自动备份。“基本”、“标准”和“高级”版数据库支持时点还原，这允许在保留期（“基本”版为 7 天，“标准”版为 14 天，“高级”版为 35 天）内进行数据库的自助还原。此外，地域复制（在“标准”和“高级”版中可用）和地域还原（在“基本”、“标准”和“高级”版中可用）在不同的级别提供了 地域冗余来针对较长时间的数据中心中断提供保护。根据你的具体备份需求或策略，例如，如果你希望将备份保留比时点还原所提供时间更长的时间，或者希望将数 据归档到本地 SQL Server 或启用到本地 SQL Server 的数据还原，则你可以考虑创建自己的备份。

  对于保留尚未承载用户创建的数据库的 SQL 数据库服务器，是否存在时间限制？

我们将会自动删除在 90 天内尚未承载用户创建的数据库的所有 SQL 数据库服务器。存储在已删除服务器中的所有用户创建的登录名都会丢失。尽管创建和维护服务器是免费的，但每台服务器都有一个与之关联的系统数据库，该数据库会占用环境中的空间。通过删除空服务器，Windows 可以更好地保持更加高效和有序的环境。

  具有高达备份存储的最大已设置数据库存储两倍的容量是什么意思？

备份存储是与用于时点还原和地域还原的自动数据库备份关联的存储。Windows Azure SQL 数据库提供了高达你的备份存储的最大已设置数据库存储两倍的容量，不需要支付额外的成本。例如，如果你有一个标准数据库实例并且设置的数据库大小为 250 GB，则会向你提供 500 GB 的备份存储并且不额外收费。如果数据库超过提供的备份存储，则可以选择与 Windows Azure 支持联系来缩短保留期，或支付按标准读访问地域冗余存储 (RA-GRS) 费率计费的额外备份存储。有关 RA-GRS 计费的更多信息，请参阅存储定价详细信息。

弹性数据库

  什么是弹性数据库？

弹性数据库适用于具有跨大量数据库的不可预测资源需求的工作负载。它们具有特定服务级别的单个数据库所拥有的全部功能，但也可通过利用弹性数据库池的资源来灵活满足不可预测的需求。

  什么是弹性数据库吞吐量单位 (DTU)？

弹性数据库吞吐量单位 (eDTU) 是可以合并到资源池的 DTU。DTU 代表数据库引擎的性能，是一个合并了 CPU、内存以及读取和写入速率的度量单位。此度量单位帮助客户评估“基本”、“标准”和“高级”级别可用的 SQL 数据库性能级别的相对性能。如需深入了解 DTU 计算包括哪些内容，请参阅 Azure SQL 数据库基准概述。

  我应在何时使用 SQL 数据库弹性数据库或单个数据库？

软件型服务 (SaaS) 提供商通常会遇到处理大量数据库的挑战。SaaS 应用程序的一个体系结构模式是为每位客户创建一个数据库。购买单独的数据库并超量设置以满足每个数据库的可变和峰值需求通常不够经济高效，而且管理多个数据库也有其自身的挑战。我们设计了弹性数据库模型来满足此需求，它还可以帮助你管理具有跨大量数据库的不可预测资源需求的工作负载。

  如何设置弹性数据库池？

弹性数据库池可以使用 Windows Azure 管理门户或通过 PowerShell 脚本创建。在文档页了解关于创建弹性数据库池的详细信息。

  如何确定应该购买的池的大小？

你可以使用 Windows Azure 管理门户中的 DTU 大小顾问，它会推荐所需的数据库和 DTU。此顾问查看要移到弹性数据库池中的数据库的历史使用量模式。这假设数据库已在 SQL 数据库中至少运行了数周。还有帮助客户基于数据库的当前使用量模式计算弹性数据库池所需 DTU 的文档。

  如何将单个数据库移到弹性数据库池中？

你可以使用 Windows Azure 管理门户将数据库移入或移出弹性数据库池。在文档页了解关于管理弹性数据库池的详细信息。

  如何将弹性数据库移出池？

你可以使用 Windows Azure 管理门户将数据库移入或移出弹性数据库池。在文档页了解关于管理弹性数据库池的详细信息。

  将数据库从单个数据库模型移到弹性数据库池需要多长时间？

将数据库从弹性数据库池移出所需的时间取决于数据库的大小。可能需要几分钟至几小时的时间，具体取决于数据库的大小。数据库在移动期间保持完全在线和可用。

  可以按何种频率调整分配给数据库池的弹性 DTU 的数量？

分配给数据库池的弹性 DTU 的数量每天可以调整一次。
